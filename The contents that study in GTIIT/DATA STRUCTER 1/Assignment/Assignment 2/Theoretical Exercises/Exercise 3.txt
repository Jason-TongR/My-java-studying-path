Exercise 3 : A data structure for anagram checking
    


    Our idea is :
        
        Why we don't use a HashMap do achieve this :

            To achieve a strict worst-case time complexity of O(m), 
                we cannot use a HashMap because hash collisions can degrade its worst-case time to O(N).
        

        Instead, we will use a Trie (Prefix Tree) :

            We will use the "id card" idea. Since the alphabet has only 26 letters, 
                we can use Counting Sort to sort any string w into a canonical string w' in O(m) time , that is , the id_card is always sorted in the English alphabet order.
                    e.g. "cba" -> its id card is "abc"             
                    e.g. "silent" -> its id card is "eilnst"

        We will insert this sorted "id card" into the Trie.





    This is the design of the data strucure : (pseudo code)




        // Define a Trie node that also stores the count of anagrams




        class TrieNode {
            TrieNode[] children = new TrieNode[26];
            int count = 0;                  
            //this is to count how many words have the same id_card (i.e. how many anagrams of this id_card we have inserted into the Trie)
        }

        public class StringSet {
            TrieNode root;

            public StringSet() {
                root = new TrieNode();
            }

        



            void insert(String w){

                // Step 1 : Compute the sorted id_card of w.

                String id_card = compute_sorted_id_card(w); 
                  

                // Step 2 : Walk down the Trie.

                TrieNode curr = root;
  
                for (char c : id_card.toCharArray()) {

                    if (curr.children[c - 'a'] == null){
                    curr.children[c - 'a'] = new TrieNode();
                    }

                    curr = curr.children[c - 'a'];

                }

                curr.count++; 
                // Mark the end of the id_card

            }

            //This is the auxiliary method to compute the sorted id_card of a string w.
            private String compute_sorted_id_card(String w) {
                
                int[] freq = new int[26];

                for (char c : w.toCharArray()) {

                    freq[c - 'a']++;

                }

                String id_card = "";

                for (int i = 0; i < 26; i++) {

                    for (int j = 0; j < freq[i]; j++) {

                        id_card += (char)(i + 'a');

                    }

                }
                
                return id_card;

            }







            int countAnagrams(String w){

                // Step 1 : Compute the sorted id_card of w.

                String id_card = compute_sorted_id_card(w); 
                


                // Step 2 : Walk down the Trie.

                TrieNode curr = root;
                
                for (char c : id_card.toCharArray()) {

                    if (curr.children[c - 'a'] == null){
                        return 0; 
                        // id_card not found
                    }

                    curr = curr.children[c - 'a'];

                }

                return curr.count;

            }

        }







    SOME COMMENT ABOUT THE CODE :
        

        1. Why compute_sorted_id_card(w) takes O(m) time?

            Instead of using a general sorting algorithm which would take O(m log m) time, 

            we use Counting Sort. We count the frequencies of characters in a fixed-size array of 26. 

            The counting step takes O(m) time, and rebuilding the string takes O(26) = O(1) time. Thus, the worst-case time complexity is strictly O(m).







        2. About the time complexity , why this two methods take O(m) time ?

            The insert method :

                    The compute_sorted_id_card(w) takes O(m) time, as explained above.
    
                    Walking down the Trie also takes O(m) time, since we process each character of the id_card once.
    
                    Therefore, the overall time complexity of the insert method is O(m).                

    
            The countAnagrams method :

                    Similar to insert, the compute_sorted_id_card(w) takes O(m) time.
    
                    Walking down the Trie also takes O(m) time.
    
                    Therefore, the overall time complexity of the countAnagrams method is O(m).
